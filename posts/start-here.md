# 从这里开始：把博客当成长期工作台

我想把这个博客做成一个长期工作台，而不是一个只在发布时才打开的展示柜。

展示柜强调完成度，工作台强调持续性。前者适合放作品，后者适合留下过程：一个问题怎么被拆开，一段经验怎么被验证，一次失败怎么变成下一次行动的起点。

## 我会写什么

这里会有三类内容：

- 前端工程里的实践笔记
- 产品体验和界面设计观察
- 独立创作、阅读和生活节奏

我希望每篇文章都尽量有一个清晰的入口：要么解释一个方法，要么留下一个判断，要么记录一个值得回看的现场。

## 为什么用 Markdown

Markdown 的好处是轻。它不会把注意力吸走，也不会强迫作者在还没想清楚时就开始排版。

写作时，我只需要打开 `posts` 文件夹，新建一篇 `.md` 文件，然后在 `posts/posts.json` 里加上标题、日期、摘要和路径。

```md
# 新文章标题

第一段正文从这里开始。
```

这就够了。真正重要的是内容的稳定积累，而不是发布流程本身有多隆重。

## 下一步

先让这个地方活起来。每周写一点，哪怕只是一个问题、一段代码、一张草图背后的判断。长期来看，博客会变成一个可搜索、可回看、也能给别人带路的知识索引。
